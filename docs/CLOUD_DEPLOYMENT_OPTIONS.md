# 클라우드 운영 방법 및 서비스 가이드

MySolar Data Collector를 클라우드에서 지속적으로 운영하기 위한 옵션 비교 가이드입니다.

---

## 🏗️ 배포 방식 비교

| 방식 | 난이도 | 비용 | 장점 | 단점 |
|------|--------|------|------|------|
| **VPS (가상서버)** | 중간 | $3-10/월 | 완전한 제어권, 유연성 | 직접 관리 필요 |
| **PaaS (플랫폼)** | 쉬움 | $0-7/월 | 자동 배포, 관리 편함 | 제한된 설정 |
| **컨테이너 서비스** | 중간 | $0-15/월 | 이식성, 확장성 | Docker 학습 필요 |

---

## 🆓 무료 옵션

### 1. Oracle Cloud Free Tier ⭐ 추천

| 항목 | 내용 |
|------|------|
| **비용** | 영구 무료 |
| **사양** | ARM 4 OCPU, 24GB RAM (매우 넉넉) |
| **장점** | 가장 좋은 무료 사양, 안정적 |
| **단점** | 가입 절차 복잡, 지역 제한 |
| **URL** | https://cloud.oracle.com |

**특징:**
- 평생 무료 ARM 인스턴스 제공
- 충분한 메모리와 CPU
- 안정적인 네트워크

---

### 2. Railway

| 항목 | 내용 |
|------|------|
| **비용** | 무료 $5 크레딧/월 (충분함) |
| **장점** | GitHub 연동 자동 배포, 매우 쉬움 |
| **단점** | 무료 한도 초과 시 중단 |
| **URL** | https://railway.app |

**특징:**
- GitHub 저장소 연결만으로 자동 배포
- 환경 변수 GUI 설정
- 로그 실시간 확인

---

### 3. Render ⭐ Background Worker 추천

| 항목 | 내용 |
|------|------|
| **비용** | Free (750시간/월) 또는 Starter ($7/월) |
| **장점** | Background Worker 지원, Docker 네이티브, 싱가포르 리전 |
| **단점** | Free 플랜은 비활성 시 중지 |
| **URL** | https://render.com |

**특징:**
- **Background Worker** 타입으로 MQTT 수집기 배포 최적
- Docker 기반 자동 빌드 및 배포
- render.yaml로 인프라 코드화 가능
- GitHub 연동 자동 배포 (autoDeploy)
- 싱가포르 리전 (한국에서 가장 가까움)

**이 프로젝트에 render.yaml 포함됨** - [상세 배포 가이드](./RENDER_DEPLOYMENT.md)

---

## 📋 일반적인 배포 순서

```
1️⃣ 클라우드 서비스 가입
         ↓
2️⃣ 서버/인스턴스 생성
         ↓
3️⃣ 서버 접속 (SSH)
         ↓
4️⃣ Node.js 설치
         ↓
5️⃣ 프로젝트 복제 (git clone)
         ↓
6️⃣ 환경 설정 (.env)
         ↓
7️⃣ PM2로 실행 (자동 재시작)
         ↓
8️⃣ 시스템 서비스 등록 (부팅 시 자동 시작)
```

## 🎯 상황별 추천

| 상황 | 추천 서비스 | 이유 |
|------|-------------|------|
| **비용 최소화** | Oracle Cloud Free Tier | 영구 무료, 높은 사양 |
| **가장 쉬운 설정** | Railway | GitHub 연동, 클릭 몇 번으로 배포 |
| **Background Worker** | Render ($7/월) | MQTT 수집기에 최적, Docker 지원 |
| **유료지만 가성비** | Vultr ($3.50/월) | 서울 리전, 저렴함 |
| **한국 리전 필수** | AWS Lightsail 또는 Vultr | 낮은 지연시간 |
| **AWS 경험 있음** | AWS Lightsail | 익숙한 환경 |

## 🔧 필수 요구사항

Solar Data Collector 운영을 위한 최소 요구사항:

| 항목 | 최소 | 권장 |
|------|------|------|
| **CPU** | 1 vCPU | 2 vCPU |
| **RAM** | 512MB | 1GB |
| **스토리지** | 10GB | 20GB |
| **OS** | Ubuntu 20.04+ | Ubuntu 22.04 LTS |
| **Node.js** | v18+ | v20 LTS |

## 📡 네트워크 요구사항

### 아웃바운드 (서버 → 외부)
- **MQTT**: 8883/TCP (HiveMQ Cloud)
- **MariaDB**: 3306/TCP (데이터베이스 서버)

### 인바운드 (외부 → 서버)
- **SSH**: 22/TCP (관리용, 필수)

## 💡 추가 고려사항

### 안정성
- PM2 프로세스 매니저 사용 (자동 재시작)
- 시스템 서비스로 등록 (부팅 시 자동 시작)
- 로그 모니터링 설정

### 보안
- SSH 키 인증 사용
- 방화벽 설정 (필요한 포트만 열기)
- 정기적인 보안 업데이트

### 모니터링
- PM2 모니터링 대시보드
- 로그 파일 관리
- 알림 설정 (선택)
