# Solar Data Collector 클라우드 마이그레이션 보고서

## 개요

MySolar 태양광 데이터 수집 시스템을 PC 환경에서 Render 클라우드 환경으로 마이그레이션하였습니다.

| 항목 | 이전 (PC) | 현재 (Render) |
|------|-----------|---------------|
| **운영 환경** | 로컬 PC | 클라우드 (싱가포르) |
| **가용성** | PC 가동 시간에 의존 | 24시간 365일 |
| **관리** | 수동 관리 필요 | 자동화된 관리 |
| **비용** | 전기료 + PC 유지비 | 월 $7 (약 9,000원) |

---

## 해결된 문제점

### 1. 데이터 수집 중단 문제

**이전 상황**:
- PC 종료 시 데이터 수집 중단
- 정전, 재부팅, Windows 업데이트 등으로 인한 서비스 중단
- 마지막 데이터 수집: 2025년 11월 14일 (약 2개월간 중단)

**해결 후**:
- 24시간 365일 무중단 데이터 수집
- 서버 장애 시 자동 재시작
- 안정적인 클라우드 인프라에서 운영

### 2. 관리 부담 감소

**이전 상황**:
- PC 상태 직접 모니터링 필요
- 오류 발생 시 수동 재시작
- 로그 파일 직접 확인

**해결 후**:
- Render 대시보드에서 상태 확인
- 자동 재시작 및 복구
- 웹 기반 로그 모니터링
- 원격 Shell 접속으로 디버깅 가능

### 3. 네트워크 안정성

**이전 상황**:
- 가정/사무실 인터넷 환경에 의존
- 공유기 재부팅, 인터넷 끊김 시 서비스 중단

**해결 후**:
- 기업용 클라우드 네트워크 사용
- 싱가포르 리전 (한국에서 가장 가까운 위치)
- 안정적인 MQTT 브로커 연결 유지

## 기술적 개선 사항

### 데이터베이스 호환성 수정

마이그레이션 과정에서 발견된 데이터베이스 호환성 문제를 해결하였습니다.

**문제점**: MariaDB의 자동 계산 컬럼(GENERATED COLUMN)에 값을 직접 삽입하려 하여 오류 발생

**해결 방법**: 자동 계산 컬럼을 제외하고 데이터 삽입하도록 코드 수정

| 자동 계산 컬럼 | 설명 |
|----------------|------|
| `solar_kwh` | 태양광 발전량 (kWh) |
| `battery_kwh` | 배터리 용량 (kWh) |
| `carbon_reduction` | 탄소 저감량 |
| `generator_status` | 발전기 상태 |

이 컬럼들은 데이터베이스에서 자동으로 계산되므로, 수집 시스템이 값을 직접 입력할 필요가 없습니다.

### 시스템 연결

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   인버터 장치    │────▶│  HiveMQ Cloud    │────▶│  Render Worker  │
│  (태양광 시스템) │     │  (MQTT Broker)   │     │  (데이터 수집)   │
└─────────────────┘     └──────────────────┘     └────────┬────────┘
                                                          │
                                                          ▼
                                                 ┌─────────────────┐
                                                 │    MariaDB      │
                                                 │  (데이터 저장)   │
                                                 └─────────────────┘
```

## 모니터링 방법

### Render 대시보드

1. https://dashboard.render.com 접속
2. `solar-data-collector` 서비스 선택
3. 확인 가능 항목:
   - **Logs**: 실시간 로그 확인
   - **Metrics**: CPU, 메모리 사용량
   - **Events**: 배포/재시작 이력
   - **Shell**: 원격 터미널 접속

### 데이터베이스 확인

```sql
-- 오늘 수집된 데이터 확인
SELECT COUNT(*) FROM raw_inverter_data
WHERE DATE(created_at) = CURDATE();

-- 최근 데이터 확인
SELECT * FROM raw_inverter_data
ORDER BY created_at DESC LIMIT 10;
```
